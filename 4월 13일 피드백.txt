[임석호]
- 함수의 인자를 잘 활용해서 코드의 재사용성을 높일 것
- 데이터를 공유하는 클래스는 따로 빼서 관리
- 클래스를 피라미드 형식으로 설계할 것!(회사의 조직도처럼)
- 매직넘버 처리 신경써서 할 것

[허진규]
- 생성자에서 메소드를 호출하지 말 것
- 매직넘버 처리에서 const? -> static으로 설정됨
                           -> constant라는 클래스를 만들어서 다같이 관리해보도록 할 것
- 매직넘버는 static으로 활용해도 좋음
  -> 변겯되지 않는 부분에 한해 static 사용해도 좋음
- string이라고 앞에 있는데 변수명에 str쓰지 말 것
- 클래스 내부에서 의미가 중복되는 메소드 네이밍을 하지 말 것
- 회원가입 중에 나가는 기능 나가도록 하는 기능도 생각할 것
- 생성자는 초기화만!
- 정규(표현)식 : 예외처리를 하는 표준 방법(문자, 숫자, 특수문자까지)
- 입력을 받을 경우 형식정도는 제시해주기
- 패키지화를 하려면 조금 더 구분해서 할 것!
